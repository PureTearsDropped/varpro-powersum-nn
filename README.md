# varpro-powersum-nn

**Sigma-Pi power-law regression with variable projection — learns a sum of product units $\hat y = \sum_k c_k \prod_i x_i^{w_{ik}}$ (learnable exponents *and* coefficients), and stays alive on poisoned data because the gradients are totalized.**

## What this is (EN)

A model that discovers **power-law (scaling) laws** from data. It fits a **sum of product units** — a Sigma-Pi form

$$\hat y = \sum_{k} c_k \prod_i x_i^{\,w_{ik}}$$

where the **exponents $w$ are learned** (each product term is a *product unit*, Durbin–Rumelhart 1989) and the **coefficients $c$ are learned** (the outer sum, a *Sigma-Pi unit*, Rumelhart–Hinton–McClelland 1986).

- **Variable projection (VarPro).** The linear coefficients $c$ are solved in closed form by least squares; only the nonlinear exponents $w$ are optimized — a hard joint fit becomes a small nonlinear one. (`separable_fit.py`)
- **Dyadic-rational prior on the exponents.** Exponents are snapped toward $\mathbb{Z}[1/2] = \{p/2^m\}$ — generated by *counting* (1) and *square root* (½) — by an F-test that snaps only when the data supports it, and **declines to snap otherwise** (hitting 12 of 17 real physics exponents; refusing on the rest rather than lying).
- **Totalized gradients (why it survives poison).** Divisions like $r^{-c}$ are kept in the *raw* domain (not hidden in log-space) and made total, so a contaminated input can never produce `NaN`/`Inf`. Crucially the **backward pass is totalized too** — the poison lives in $-\text{num}/\text{den}^2$ — and the resulting **saturation flags double as an automatic outlier detector**, cleaning bad rows out of the loss during training. (`physics_tot_train.py`)

### How it differs from a textbook product unit

The classic Durbin–Rumelhart product unit lives in **log-space** ($y = e^{\sum p\log|x|}\cos\pi V$): it turns products into sums, never divides, and carries sign through $\cos\pi V$ (which collapses to 0 at half-integer $V$). This model does the opposite — it **keeps the division explicit and totalizes it** — which is exactly why total arithmetic and gradient flags have something to bite on.

### Reproduce

```bash
python3 -m venv .venv && . .venv/bin/activate && pip install -r requirements.txt
python total_bench.py          # VarPro power-sum fit over 12 laws, CPU/numpy (no GPU needed)
python physics_tot_train.py    # totalized-gradient demo (needs CUDA; falls back to CPU)
```

**Measured (`physics_tot_train.py`, RTX 5090)** — learning $F = G\,m_1 m_2 / r^2$ with 4 poisoned rows (r≈1e-20) in 260, 8 seeds:

| arm | dead | exponent error (median / worst) | NaN grads | poison flagged | false pos |
|---|---|---|---|---|---|
| total arith, clean (baseline) | 0/8 | 0.009 / 0.021 | 0 | — | — |
| **IEEE float32** | **8/8** | — (wiped out) | **16000** | — | — |
| total arith (no mask) | 0/8 | 2.579 / 2.677 | 0 | 4/4 | 0 |
| **total arith + flag mask** | **0/8** | **0.009 / 0.021** | 0 | 4/4 | **0** |

IEEE dies on every seed; totalized arithmetic survives, and the gradient flags recover the clean-data accuracy exactly while naming all 4 poisoned rows with zero false positives.

### Files

- `separable_fit.py` — VarPro engine (numpy/scipy): power-sum basis, closed-form linear solve, dyadic F-test snapping.
- `total_arith.py` — numpy reference of the total-arithmetic convention (`a/0=0`, `±MIN`/`±MAX`, no `NaN`/`Inf`).
- `total_bench.py`, `total_bound.py` — power-sum fitting benchmarked under total arithmetic.
- `physics_tot_train.py` — the totalized-gradient training demo (custom `SatDiv` autograd, gradient flags, self-cleaning loss).

## これは何か（JP）

データから**べき乗則（スケーリング則）**を見つけるモデル。**product unit の和**（Sigma-Pi 形）

$$\hat y = \sum_{k} c_k \prod_i x_i^{\,w_{ik}}$$

を当てる。**指数 $w$ を学習**（各項が product unit・Durbin–Rumelhart 1989）、**係数 $c$ を学習**（外側の和・Sigma-Pi ユニット）。

- **VarPro（変数射影）**: 線形係数 $c$ は lstsq で閉形式、非線形の指数 $w$ だけ最適化。
- **指数への二進有理事前**: $\mathbb{Z}[1/2]=\{p/2^m\}$（生成元＝「数える」と「平方根」）にF検定で丸める。支持がなければ丸めず人に見せる（実在物理指数17個中12個的中）。
- **勾配の全域化（毒に強い理由）**: $r^{-c}$ の除算を*生の領域*のまま全域化。**逆伝播も全域化**（毒は $-\text{num}/\text{den}^2$ に住む）し、**飽和フラグが外れ値検出器を兼ねて**汚染行を損失から自動除外。

**標準の product unit との違い**: 古典（D&R）はlog空間 $y=e^{\sum p\log|x|}\cos\pi V$ で、積を和にして割り算しない。本モデルは逆に**割り算を残して全域化**する——だから全域算術と勾配フラグが効く。

## Roadmap

The three repositories below are, in effect, **three backends of one total-arithmetic contract** (CPU / GPU / hardware). A planned next step is a **pluggable backend** so this model can run its total arithmetic on any of them:

- **[total-arith-cuda](../../total-arith-cuda)** — the GPU (torch/CUDA) implementation.
- **[total-arith-hardware](../../total-arith-hardware)** — the gate → SystemVerilog → FPGA implementation.

Today `physics_tot_train.py` carries its own inline torch total-arithmetic; unifying these behind one interface is future work.

[total-arith-cuda](https://github.com/PureTearsDropped/total-arith-cuda) (verify with `diff`;
same pattern as robust-attitude-control). The forward totalization in `physics_tot_train.py`
now DELEGATES to the audited `_sat` (5 external audit rounds + semantic flag oracle) instead
of a hand-rolled copy — and the switch was motivated by a real hole the conformance check
found: **the old hand-rolled `sat` passed NaN through** (`a > MAX` is False for NaN), so a
`0×inf` upstream would have violated the "never NaN" banner. The audited core names it:
NaN → (0, no-bound+SUNK). What stays hand-written on purpose: `SatDiv`'s **totalized
backward** (gradient flags) — this repo's contribution, outside the core's scope; its value
semantics are checked against `cuda_total.tot_div` (4096 cases incl. 585 divisions by zero,
exact match). Run `python conformance_tot.py` for the permanent regression; the full 3-arm
experiment reproduces identically after the switch (IEEE 8/8 dead / TOT+mask median 0.009,
flags 4/4 hit, 0 false).

### `implicit_discovery.py` — implicit law discovery (Python twin of total-arith-cuda's julia/Discovery.jl)

Laws as `Σc·(monomials/derivatives) = 0`, solved as the null space (one SVD, gradient-free,
seed-free). Born from the observation that y=f(x) explicit regression hits a composition
wall at relativity's γ=(1−v²)^{-1/2}, while the implicit form makes it polynomial. Pipeline:
audited `Tot` entry names poison (flags decide, float64 values keep precision) → flag-row
exclusion → library → SVD → gap judgment → tautology detector (evaluate the found law on
physics-free random data — still zero ⟹ learned nothing). Regression-tested trophies:
E²=m²+p² and γ²(1−v²)=1 at ±1.000000; poisoned hydrogen where naive SVD dies → E=−0.500000
rescued; honest refusal on lawless data; complex support — the free Schrödinger equation's
imaginary unit discovered as a coefficient (ψ_xx/ψ_t = −0.5003i).
